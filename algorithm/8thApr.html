# 分而治之

一种著名的递归式解决问题的方法。(divide and conquer,D&C)

工作原理：

  1. 找出简单的基线条件
  2. 确定如何缩小问题的规模，使其符合基线条件

编写涉及数组的递归函数时，基线条件通常是数组为空或只包含一个元素。

举几个例子,具体是什么可以看一下P47

def sun(list):
    if list == []:
        return 0
    else return list[0] + sun(list[1:])

def count(list):
    if list == 0:
        return 0
    else return 1 + count(list[1:])

def max(list):
    if len(list) == 2:
        return list[0] if list[0] > list[1] else list[1]
    sub_max = max(list[1:])
    return list[0] if list[0] > sub_max else sub_max

二分查找也是一种分而治之算法。它的基线条件是数组只包含一个元素。

如果要查找的值与这个元素相同，就找到了！否则，就说明他不在这个数组里。

在二分查找的递归条件中，你把数组分成两半，将其中一半丢弃，并对另一半执行二分查找。

# 快速排序

  1. 首先从数组中选择一个元素，这个元素被称为基准值(pivot)
  2. 接下来找出比基准值小的元素以及比基准值大的元素。这称为分区。现在你有：
  • 一个由所有小于基准值的数字组成的子数组；
  • 基准值；
  • 一个由所有大于基准值的数组组成的子数组；
  1. 对这两个子数组进行快速排序
def quicksort(array):
    if len(array) < 2:
        return array #基线条件：为空或只包含一个元素的数组是有序的
    else : 
        pivot = array[0] #递归条件
        less = [i for i in array[1:] if i <= pivot] #由所有小于等于基准值的数组组成的子数组
        greater = [i for i in array[1:] if i > pivot] #由所有大于基准值的数组组成的子数组

        return quicksort(less) + [pivot] + quicksort(greater)
print (quicksort([10,5,2,3]))

# 再谈大O

快速排序在最糟糕的情况下，运行时间为O(n^2)。再平均情况下，快排的运行时间为O(nlogn)

快速排序的性能高度依赖于你选择的基准值，假设你总是讲第一个元素用作基准值，且要处理的数组是有序的。

由于快速排序算法不检查输入的数组是否有序，因此它依然尝试对其进行排序。

同理，在中间是最佳排序。最糟情况下，栈长为O(n),最佳情况下，栈长为O(logn).

再来看看层数，第一层，将一个元素用作基准值，并将其他元素划分到两个子数组中。这涉及数组中的全部8个元素，因此该操作时间为O(n)。

但实际上，在调用栈的每层都涉及O(n)个元素。

因此，完成每层所需的时间都为O(n)

在这个示例中，层数为O(logn)用技术术语来说，调用栈的高度为O(logn)，而每层需要的时间为O(n)

所以最佳用了O(nlogn)，最糟则是O(n^2)。

这里要补充说明一下，最佳情况也是平均情况，只要每次随即地选择一个数组元素作为基准值，快速排序的平均运行时间就将为O(nlogn)

快速排序是最快的排序算法之一，也是D&C的典范。

# 总结

  • D&C将问题逐步分解，使用D&C处理列表时，基线条件很可能是空数组或只包含一个元素的数组。
  • 实现快速排序时，请随机的玄色用作基准值的元素。快速排序的平均运行时间为O(nlogn)
  • 大O表示法中的常量有时候事关重大，这就是快速排序比合并排序快的原因所在。
  • 比较简单查找和二分查找时，常量几乎无关紧要，因为列表很长时，O(logn)的速度比O(n)快得多
