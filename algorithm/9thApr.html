# 散列函数

散列函数是这样的函数，即无论你给他什么数据，他都还你一个数字。

散列函数其实必须满足一些要求

  • 它必须是一致的。假如你输入apple得到的是4，那么每次输入apple，得到的都必须是4
  • 他应将不同的输入映射到不同的数字。(最理想情况)

散列函数这么具体的原因如下

  • 总是将同样的输入映射到相同的索引。
  • 将不同的输入映射到不同的索引
  • 知道数组有多大，只返回有效索引

python中的字典，就是一个散列表！散列表由键和值组成。

# 实现

cache就是用散列表储存的，用python实现如下

cache = {}
def get_page(url):
    if cache.get(url):
        return cache[url] #返回缓存的数据
    else:
        data = get_data_from_server(url)
        cache[url] = data   #将数据保存到缓存中
        return data

# 小结

散列表适用于：

  • 模拟映射关系
  • 防止重复
  • 缓存/记住数据

# 冲突

简单来说，就是给两个键分配的位置相同

而处理冲突最简单的方式，就是如果两个键映射到了同一个位置，那么就在这个位置存储一个链表。

# 性能

在平均情况下，散列表执行各种操作的时间都为O(1),这被称为常量时间。

在最高情况下，散列表所有操作的运行时间都为O(n)，线性时间。

散列表(平均) 散列表(最糟) 数组 链表
查找 O(1) O(n) O(1) O(n)
插入 O(1 ) O(n) O(n) O(1)
删除 O(1 ) O(n) O(n) O(1)

在平均情况下，散列表的查找(获取给定索引处的值)速度和数组一样快，而插入和删除速度与链表一样快

但在最糟情况下，散列表的各项操作的速度都很慢

因此，在使用散列表时，避开最早情况至关重要

因此要有

  • 较低的填装因子
  • 良好的散列函数

# 填装因子

散列表的填装因子很容易计算

散列表包含的元素数/位置总数

填装因子的度量的是散列表中有多少位置是空的

一个不错的经验规则是：一旦填装因子大于0.7，就调整散列表的长度

# 总结

  • 可以结合散列函数和数组来创建散列表
  • 冲突很糟糕，应使用可以最大限度减少冲突的散列函数
  • 散列表的查找，插入和删除速度都非常快
  • 散列表适用于模拟拟合关系
  • 一旦填装因子超过0.7，就该调整散列表长度
  • 散列表可用于缓存数据
  • 散列表非常适合用于防止重复。
