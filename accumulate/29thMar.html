# Call() Apply() and Bind()

这三个函数都是function自带的，都是为了改变函数体内部 this 的指向。

apply 、 call 、bind 三者第一个参数都是 this 要指向的对象，也就是想指定的上下文。

apply 、 call 、bind 三者都可以利用后续参数传参。

# Call()

为什么先记录call呢，我个人觉得call最好理解。

var a = {
    user:"Roy",
    fn:function(){
        console.log(this.user); //Roy
    }
}
var b = a.fn;
b.call(a);

如果我们不用call方法，a.fn指向的是windows(this指的)。由于windows没有user这个类型，consol.log()显示undefined。

通过call方法，给第一个参数添加要把b添加到哪个环境中，简单来说，this就会指向那个对象。这样就会从windows,变成a,输出Roy。

call方法除了第一个参数以外还可以添加多个参数，如下：

var a = {
    user:"Roy",
    fn:function(e,ee){
        console.log(this.user); //Roy
        console.log(e+ee); //3
    }
}
var b = a.fn;
b.call(a,1,2);

apply()和call()可以理解为功能类似，他们都可以改变this的指向，这里不再详细解释。

但是不同的是，apply()和call()的参数是不同的，call()的元素只能是元素，而apply()的可以为数组。

var a = {
    user:"Roy",
    fn:function(e,ee){
        console.log(this.user); //Roy
        console.log(e+ee); //11
    }
}
var b = a.fn;
b.apply(a,[10,1]);
/*或者
var arr = [500,20];
b.apply(a,arr);
*/

# Bind()

还是那句话，他们三个都能改变this的指向。

让我们来试一下。

var a = {
    user:"Roy",
    fn:function(){
        console.log(this.user);

var b = a.fn;
b.bind(a);

然而会发现并没有打印this.user,为什么呢？因为bind()不是像另两种方法那样可以即刻改变函数this的指向，而是返回了一个函数。

var a = {
    user:"Roy",
    fn:function(){
        console.log(this.user); //Roy
    }
}
var b = a.fn;
var c = b.bind(a);

ok，同样bind也可以有多个参数，并且参数可以执行的时候再次添加，但是要注意的是，参数是按照形参的顺序进行的。

var a = {
    user:"Roy",
    fn:function(e,d,f){
        console.log(this.user); //Roy
        console.log(e,d,f); //10 1 2
    }
}
var b = a.fn;
var c = b.bind(a,10);
c(1,2);

# 总结

call和apply都是改变上下文中的this并立即执行这个函数，bind方法可以让对应的函数想什么时候调就什么时候调用，并且可以将参数在执行的时候添加，这是它们的区别，根据自己的实际情况来选择使用。
